    {% trans "Exploring Transactions" %}

    {% trans "Let's go over some basic principals that Jade uses thoughout the system. A sale is just one type of Transaction that Jade handles some other examples are: Payments, Returns, Physical Counts and Production. All transactions have some common traits." %}

    {% trans "Document Number" %}

    {% trans "Document Numbers are used to group transactions. A Sale, Payment, Return and Garantee are all different types of Transactions, but we can group them together by giving them all the same document number. But we must be careful that we keep them unique. For example: if I were to sell a block to Susan and another to Fred, if I give them both the same document number, they would both show up on the same receipt ... probably not what we want. Stick with a simple numbering scheme and you won't have any problem." %}

    {% trans "Entries" %}

    {% trans "Transactions a made up of entries. Entries are the building blocks of Jade's accounting system. Jade makes these entries for us automaticly, and we can't change them directly, but if you want to see exactly what is going on behind the scenes, click on the list icon and Jade will show you the entries for the transaction in question. You'll be happy to know that Jade treats inventory just as an accountant treats money. It is all done with double-entry bookkeeping methods so we can be sure that as long as we enter the information, it's all accounted for." %}

    {% trans "Activate/Deactivate " %}

    {% trans "If you want Jade to 'ignore' the price or cost of a certain transaction, click on the light bulb and it will turn off . As long as the light bulb is off, the value of the transaction will not be taken into account in the accounting aspect of the system. You might want to do this if the client wants a quote but is not sure yet if he really wants to make the purchase. Leave the light bulb off until he decides, then you can either delete the transaction or turn the light bulb back on. As a matter of fact, if you click on the printer icon for a sale, and Jade notices that all of the transactions with that document number are deactivated, it will produce a quote for you instead of the normal receipt." %}

    {% trans "Delivered/Undelivered " %}

    {% trans "Just as the deactivate option controls whether or not the transaction is taken into consideration in the accounting, the Devered option controls whether or not the quantity is applied to the inventory. For example: perhaps a purchase has already been made, but has not yet been received. How do we keep the quantity from showing up in our inventory? Click on the Delivered icon which is normally a box with a green check and it will change the icon to a box with a red 'x' . Now the items will not appear in our inventory until we mark it as delivered." %}

    {% trans "Details " %}

    {% trans "Sometimes, to serve you faster, Jade will only show us a preview of a transaction, the date, account, item, quantity, and value, and it will leave out some of the details like the cost, tax, and discount. If you want to see those details for a certain transaction, just click on the white arrow and Jade will fetch all of those details for you." %}

    {% trans "Edit , Save , & Cancel " %}

    {% trans "If you see that you made a mistake, don't worry, just click on the icon of a paper and pencil . This icon allows you to edit the sale again. When you are finished making your changes, click on the diskette and your changes will be saved. If you don't want to save your changes click on the red 'X' to cancel any changes made." %}
