    {% trans "Adding Clients and Vendors" %}

    {% trans "To add a client or a vendor, click on the 'Accounts' tab and then on the Account type, either 'Client' or 'Vendor' and then click 'New'. Type in the persons name and click 'Save'. The alternative is to simply create a sale or a purchase for the person. The person will automaticly be added to the database. See Chapter 2: Sales Now let's go over some of the other fields on this page. Since the fields for vendors and clients are the same. We'll consider both situations at the same time." %}

    {% trans "Account Number" %}

    {% trans "The account number is very important because it is used to calculate the balance on an account. For example: By default Jade comes with a 'Clients' account which has an account number '0103'. The balance for that account will not only include transactions with that account number but for any account that starts with '0103'. So if we want our new client's balance to be included in the total, our clients account number must start with '0103'. By default, if we create a client without specifing an account number, Jade will take care of things and assign the client an appropriate account number. The same principle applies to Vendors." %}

    {% trans "Multiplier" %}

    {% trans "The multiplier is a peice of accounting and is either Credit or Debit. Clients will always be Debit accounts and Vendors will always be Credit accounts. The reasons why is beyond the scope of this manual and is best explained by an accountant. For a little more information see the section 'Creating an account' later in this chapter." %}

    {% trans "Account Group" %}

    {% trans "The account's Account Group determines where Jade stores the values of the revenue, discounts, returns, etc. for the client. For more information about Account Groups see Chapter 9: Account, Receipt, & Price Groups" %}

    {% trans "Price Group" %}

    {% trans "This is the name of the Price Group the client or Vendor belongs to. When products are sold to this person, prices will be retreived for this price group. By default Jade comes with two price groups: Retail and Wholesale." %}

    {% trans "Credit Days" %}

    {% trans "When a sale is made without a payment, the sale will appear in the Unpaid Sales section. If the number of days specified by Credit Days has passed since the Sale was made the Transaction will appear in the Overdue section" %}

    {% trans "Balance" %}

    {% trans "The total amount that this client owes us, or in the case of a Vendor, the total amount that we owe them. Extra note: that this is the sum of the values of all of the transactions with account numbers starting whith this persons account number" %}

    {% trans "User" %}

    {% trans "The name of the user responsible for dealing with this client or vendor." %}

    {% trans "Other fields" %}

    {% trans "The other fields are not used by Jade and are optional. The information from these fields may be used on quotes and receipts. See Chapter 11: Reports" %}

    {% trans "Unpaid and Overdue Transactions" %}

    {% trans "When a sale is made without a payment, the sale will appear in the Unpaid Sales section. If the number of days specified by Credit Days has passed since the Sale was made the Transaction will appear in the Overdue section. To change these entries, click on the document number of the transaction in question." %}

    {% trans "Creating Accounts" %}

    {% trans "While the accounts you use most of the time may be Clients and Vendors, Jade also uses generic Accounts to organize your accounting. Although Jade comes with a complete collection out of the box, you may want to add some accounts of your own. For example: You may have two taxes that your government requires, and you may need to keep their values separate. In this case you would create two accounts for the new tax. One for purchases and one for sales. (Unless of course you prefer to lump the two values together in which case you would only need one new account.) First determine under which branch of the accounting structure the new account should be included for example, our sales tax would fall under 020201 - Sales Taxes. Then determine the next available account number for the branch. If we're starting with the default Jade installation, the next available number for our example would be 02020102 since 02020101 is already in use. Now the easy part: From the Accounts page, click New and type in the Name of the new account and the new account number. Make sure you make the multiplier to match the parent account. Since 020201 - Sales Taxes is a Credit account, we'll make our new account Credit as well. When you're ready click Save. See Chapter 10: Price Groups & Taxes for more information about creating taxes." %}
